# QA Automation Roadmap (Лето - Зима 2026)

Этот репозиторий — моя база знаний и портфолио в процессе перехода в QA. План разбит на небольшие этапы для комфортного совмещения с работой.

## 🛠 Этап 1: База ручного тестирования
- [ ] Что такое баг и анатомия идеального баг-репорта (Bug Report).
- [ ] Жизненный цикл бага (от находки до релиза фикса).
- [ ] Что такое тест-кейс (Test Case) и чек-лист.
- [ ] Техники тест-дизайна: как тестировать минимум, проверяя максимум (Эквивалентные классы, Граничные значения).

## 🌐 Этап 2: Веб-технологии и API
- [ ] Как работает интернет: Клиент-серверная архитектура.
- [ ] HTTP-запросы: разница между GET, POST, PUT, DELETE.
- [ ] Статус-коды серверов (200, 400, 404, 500 и т.д.).
- [ ] Формат обмена данными JSON.
- [ ] Практика: отправка запросов через Postman / Insomnia.

## 🗄️ Этап 3: Базы данных
- [ ] Что такое реляционные базы данных.
- [ ] Базовые SQL-запросы: SELECT, INSERT, UPDATE, DELETE.
- [ ] Фильтрация данных (WHERE) и объединение таблиц (JOIN).
- [ ] Как QA использует SQL в реальной работе.

## 🤖 Этап 4: Автоматизация и Playwright
- [ ] Выбор языка для автотестов (вспоминаем базовую логику скриптинга).
- [ ] Что такое DOM-дерево и как находить элементы на странице (Локаторы).
- [ ] Знакомство с фреймворком Playwright.
- [ ] Написание первого UI-автотеста.

## 🎯 Этап 5: Трудоустройство
- [ ] Оформление резюме.
- [ ] Подготовка к типичным вопросам на собеседовании.
