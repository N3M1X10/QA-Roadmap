# Техники тест-дизайна 🎯

Позволяют сократить количество проверок, сохраняя высокое качество тестирования.

### 1. Эквивалентное разделение (Equivalence Partitioning)
Разделение всех входных данных на группы, для которых поведение системы одинаково. Из одной группы тестируется только одно значение.

*Пример для диапазона [18 - 65]:*
* Невалидный класс (меньше 18) -> Тест: `10` (Отказ)
* Валидный класс (от 18 до 65) -> Тест: `30` (Успех)
* Невалидный класс (больше 65) -> Тест: `80` (Отказ)

### 2. Анализ граничных значений (Boundary Value Analysis)
Тестирование стыков между классами эквивалентности, так как на границах диапазонов чаще всего случаются ошибки в коде. Проверяется сама граница и значения на один шаг в обе стороны.

*Пример для диапазона [18 - 65]:*
* Граница 18 -> Тесты: `17`, `18`, `19`
* Граница 65 -> Тесты: `64`, `65`, `66`
