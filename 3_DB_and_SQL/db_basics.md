# Работа с базами данных и SQL 🗄️

## Реляционные базы данных (РБД)
Способ организации данных, где вся информация хранится в виде взаимосвязанных таблиц (строки и столбцы). 

* **Зачем это QA:** Проверять бэкенд на честность. Убеждаться, что данные после отправки через UI или API корректно сохраняются, обновляются или удаляются из базы данных.

---

## Базовые SQL-запросы: Получение данных

**SQL (Structured Query Language)** — язык запросов к базам данных.

### Команда SELECT
Используется для чтения и извлечения данных из таблиц. Никак не изменяет саму базу.

* **Выбрать все данные из таблицы:**
```sql
SELECT * FROM orders;
```

* **Выбрать конкретные столбцы (вертикальная фильтрация):**
```sql
SELECT id, status, delivery_address FROM orders;
```

---

## Изменение данных: INSERT, UPDATE, DELETE

### 1. INSERT INTO (Добавление строк)
```sql
INSERT INTO table_name (column1, column2) VALUES (value1, value2);
```

### 2. UPDATE (Изменение строк)
```sql
UPDATE table_name SET column_name = new_value WHERE condition;
```
*Важно:* Всегда нужно указывать условие WHERE, иначе изменятся данные во всей таблице.

### 3. DELETE (Удаление строк)
```sql
DELETE FROM table_name WHERE condition;
```
*Важно:* Без условия WHERE таблица будет полностью очищена.

## Фильтрация данных: Оператор WHERE и условия

Чтобы не выгребать из базы миллионы строк, используется оператор `WHERE`. Он фильтрует строки по заданным критериям.

### 1. Базовые операторы сравнения
* `=` (равно), `<>` или `!=` (не равно)
* `>`, `<`, `>=`, `<=` (больше, меньше, равно)

```sql
SELECT * FROM couriers WHERE transport = 'bicycle';
```

### 2. Логические связки (AND, OR)
Позволяют объединять несколько условий в один запрос.
* `AND` — строка попадет в результат, только если выполняются **все** условия одновременно.
* `OR` — строка попадет в результат, если выполняется **хотя бы одно** из условий.

```sql
SELECT * FROM orders WHERE status = 'created' AND hub_id = 7;
```

### 3. Поиск по списку значений (IN)
Конструкция, которая заменяет кучу длинных условий с `OR`, если нужно проверить совпадение с конкретным списком.

```sql
SELECT * FROM orders WHERE status IN ('canceled', 'returned', 'failed');
```

### 4. Поиск по шаблону (LIKE)
Используется для гибкого поиска по тексту. Специальный знак `%` работает как маска и означает «любое количество любых символов».
* `LIKE 'Алекс%'` — найдет Алексея, Александра, Алекс.
* `LIKE '%Ленина%'` — найдет любой адрес, в котором упоминается улица Ленина.

```sql
SELECT id, name, phone FROM couriers WHERE name LIKE 'Ива%';
```
