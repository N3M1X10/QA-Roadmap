# Практика автоматизации на Playwright

Проект содержит тренировочные E2E-тесты, написанные на JavaScript + Playwright. Архитектура построена с учетом постепенного перехода от линейных скриптов к паттерну Page Object Model (POM).

## Трект обучения и интеграции

### 1. Базовые сценарии и локаторы
- [ ] Инициализация браузера и базовая навигация.
- [ ] Работа с локаторами (CSS-селекторы, текстовые узлы, роли).
- [ ] Взаимодействие с UI-элементами (ввод данных, клики, состояния).
- [ ] Написание сквозного (E2E) теста авторизации и покупки на SauceDemo.

### 2. Архитектура и оптимизация
- [ ] Рефакторинг линейных автотестов.
- [ ] Внедрение Page Object Model (изоляция логики страниц).
- [ ] Разделение тестовых данных и конфигурации.

### 3. Работа с API и фикстурами
- [ ] Использование встроенных фикстур Playwright.
- [ ] Перехват и мокинг сетевых запросов.
- [ ] Безопасное хранение учетных данных через `.env`.

### 4. Инфраструктура (CI/CD и отчетность)
- [ ] Подключение и настройка Allure-репортера.
- [ ] Создание пайплайна GitHub Actions для автоматического тестирования при Pull Request.
