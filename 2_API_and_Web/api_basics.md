# Веб-технологии и API 🌐

## Клиент-серверная архитектура
Фундамент работы интернета. Взаимодействие всегда происходит по принципу: **Запрос (Request) -> Ответ (Response)**.

* **Клиент (Client):** Устройство или программа на стороне пользователя (браузер, мобильное приложение, десктопная программа). Отвечает за отображение интерфейса и отправку запросов.
* **Сервер (Server):** Удаленный мощный компьютер, который хранит данные, обрабатывает логику работы системы и возвращает клиенту нужный результат.

*Пример:* Приложение погоды на телефоне (Клиент) отправляет запрос на сервер метеостанции. Сервер находит данные для текущего города и отправляет температуру обратно на телефон (Ответ).

---

## HTTP-методы (GET, POST, PUT, DELETE)
Каждый запрос клиента содержит метод — команду, которая говорит серверу, какое действие нужно выполнить.

* **`GET`** — Получение данных. Безопасен для системы (не меняет данные на сервере).
  * *Пример:* Посмотреть список товаров, открыть страницу профиля.
* **`POST`** — Создание новых данных.
  * *Пример:* Отправка формы регистрации, создание нового заказа, загрузка фотографии.
* **`PUT`** — Обновление/модификация существующих данных.
  * *Пример:* Изменение имени в профиле, редактирование текста комментария.
* **`DELETE`** — Удаление данных.
  * *Пример:* Удаление товара из корзины, удаление аккаунта.

## Статус-коды серверов (HTTP Status Codes)
Трёхзначный код в ответе сервера, который показывает результат обработки запроса.

* **`2xx` — Успех (Success)**
  * `200 OK` — Запрос успешно выполнен.
  * `201 Created` — Запрос выполнен, внутри базы данных создан новый объект (пользователь, заказ).
* **`3xx` — Редирект (Redirection)**
  * `301 Moved Permanently` — Запрашиваемый ресурс навсегда переехал на другой URL.
* **`4xx` — Ошибка клиента (Client Error)**
  * `400 Bad Request` — Неверный запрос (сервер не может обработать из-за ошибок в данных клиента).
  * `401 Unauthorized` — Требуется авторизация (клиент не залогинен).
  * `403 Forbidden` — Доступ запрещен (логин верный, но прав на это действие нет).
  * `404 Not Found` — Ресурс/страница не найдены (ошибка в адресе).
* **`5xx` — Ошибка сервера (Server Error)**
  * `500 Internal Server Error` — Внутренняя ошибка сервера (упал код, сломалась логика).
  * `504 Gateway Timeout` — Тайм-аут шлюза (сервер не ответил за отведенное время, сайт "лежит").

## Структура HTTP-запроса и ответа
Обмен данными между клиентом и сервером состоит из строго структурированных блоков метаданных и полезной нагрузки.

### Из чего состоит Запрос (Request):
* **Стартовая строка:** Метод, путь (URI) и версия HTTP (например, `GET /index.html HTTP/1.1`).
* **Параметры:**
  * *Path-параметры:* Часть пути URL для идентификации ресурса (`/api/users/105`).
  * *Query-параметры:* Передаются после знака `?` для фильтрации/сортировки (`/api/users?role=courier&page=2`).
* **Заголовки (Headers):** Метаданные (служебная информация).
  * `Content-Type` — формат отправляемых данных (например, `application/json`).
  * `Authorization` — токен или данные для проверки прав доступа.
* **Тело запроса (Body):** Сами данные (payload). Используется в `POST` и `PUT` запросах для отправки тяжелых объектов.

### Из чего состоит Ответ (Response):
* **Строка статуса:** Версия HTTP и статус-код (например, `HTTP/1.1 200 OK`).
* **Заголовки ответа:** Служебная информация от сервера (тип контента, кодировка, дата).
* **Тело ответа (Body):** Данные, которые вернул сервер (HTML-страница, файл, картинка или JSON-структура).

## Что такое REST API и формат JSON

### REST API (базовое понятие)
Архитектурный стиль взаимодействия между клиентом и сервером. В REST:
* Каждому ресурсу (пользователь, заказ, товар) соответствует свой URL (например, `/api/v1/users`).
* Для управления ресурсами используются стандартные HTTP-методы CRUD (`GET`, `POST`, `PUT`, `DELETE`).
* Сервер не хранит состояние клиента (каждый запрос автономен и содержит все данные для проверки прав).

### Формат JSON (JavaScript Object Notation)
Текстовый формат обмена данными, основанный на синтаксисе объектов JavaScript. Является стандартом для REST API.

**Критические правила синтаксиса JSON:**
1. Ключи и строковые значения *всегда* оборачиваются в двойные кавычки `""` (одинарные запрещены).
2. Запрещены "висячие" запятые после последнего элемента в объекте или массиве.
3. Разрешенные типы данных: String, Number, Object, Array, Boolean, Null.
